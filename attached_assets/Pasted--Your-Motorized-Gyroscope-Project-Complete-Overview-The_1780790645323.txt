## The Hardware (What You're Building)

### The Rotating Part
- **Flywheel:** A carefully balanced disk (~0.86-1.2 kg) that stores rotational energy
- **Shaft:** 8mm steel rod for strength and rigidity
- **Bearings:** RS608 high-speed ball bearings for low friction
- **Target Speed:** 8,000-12,000 RPM (really fast!)

### The Power System
- **Motor:** 950 KV brushless motor (found locally in Zambia!)
- **ESC:** Electronic Speed Controller to manage power flow
- **Power Supply:** 12-24V AC-to-DC wall supply (96W) - this is wall-powered, so it can run forever
- **Motor Coupling:** Flexible connector between motor and flywheel shaft

### The Sensors (For Monitoring)
- **MPU6050:** 6-axis IMU (accelerometer + gyroscope) - measures tilt angle
- **IR Speed Sensor:** Counts flywheel rotations to measure RPM
- **Temperature Sensor:** Monitors heat buildup (prevents overheating)
- **SD Card Module:** Records all data for analysis

### The Brain
- **ESP32 DevKit:** Microcontroller that runs the control algorithm, reads sensors, and drives the motor

---

## The Software (How It Works)

### 1. **Control Algorithm** (MATLAB Simulink)
You built a **feedback control system** that:
- Reads the tilt angle from the MPU6050 sensor
- Compares it to the desired level (0°)
- Adjusts motor power to keep the gyroscope balanced
- Uses a PID-like controller (proportional-integral-derivative tuning)

**The Physics:** When a spinning gyroscope is tilted, it naturally wants to precess (wobble sideways). Your control system counteracts this by adjusting the motor speed.

### 2. **ESP32 Firmware** (C++ code)
- Runs the control algorithm in real-time (~1000 times per second)
- Reads data from sensors via I2C protocol
- Sends PWM signals to the ESC to control motor speed
- Logs data to SD card for later analysis

### 3. **Real-Time Visualization** (Python Dash App)
You just built this! It:
- Reads sensor data from ESP32 via serial port (COM5 @ 115200 baud)
- Shows **3 live graphs** (X tilt, Y tilt, Z rotation) in a web browser
- Updates every 100ms without page refreshes (true real-time streaming)
- Configurable via .env file (easy to change COM port, baud rate, etc.)
