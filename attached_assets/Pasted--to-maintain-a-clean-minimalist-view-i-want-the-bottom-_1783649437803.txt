Critical findings:

Command gate coverage is largely correct for terminal input + terminal-bus paths: SystemTerminal.submitCommand enforces isCommandAllowed before send (src/components/SystemTerminal.tsx:284-294), and dashboard s/x/h/l/z/w shortcuts plus bottom-nav actions route through sendTerminalCommand (src/pages/dashboard.tsx:568-581, src/components/BottomNavBar.tsx:55-72).
IR and other non-terminal paths do bypass command-gate by design and rely on separate guards (src/components/SessionControl.tsx:205-287; dashboard Esc/session-start handlers at src/pages/dashboard.tsx:498-556). This is functional but creates policy drift risk because gating logic is now duplicated across modules.
The new optimistic throttle readout is materially inaccurate: throttleUs is only adjusted for keyboard h/l in dashboard (src/pages/dashboard.tsx:579-580). Bottom-nav h/l, terminal-typed h/l / CMD:SPEED_*, and IR/other hardware-side changes do not update it (BottomNavBar.tsx:63,71; SystemTerminal.tsx send path), so the displayed µs can diverge from actual motor state.
Keyboard q session-start handler duplicates CH+ logic and lacks in-flight dedupe (src/pages/dashboard.tsx:523-555), so repeated key presses can trigger multiple concurrent POST /api/session calls before session state flips. This same pattern exists in IR CH+ (SessionControl.tsx:230-264), confirming drift/consistency risk from duplicated start logic.
Auto-hide nav has accessibility concerns: collapsed reveal depends on hover/low-visibility pull-tab behavior (BottomNavBar.tsx:90-100,102-109), with no explicit keyboard/focus affordance indicators or aria labeling for the tab control.
Security: none observed (no serious auth/injection/secret handling regressions in these changes).

Next actions:

Introduce a single shared startSession action (used by CH+, keyboard q, and start button) with a pending/lock guard to prevent duplicate requests.
Make throttle display explicitly “estimated” and update from all command origins (terminal submit observer + bottom-nav + keyboard), or remove live µs readout until hardware truth is available.
Improve bottom-nav accessibility: add clear focus-visible behavior + aria-label for pull-tab and ensure reveal/collapse is fully operable without hover.
